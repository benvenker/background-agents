# Epic: `EXT2-EPIC-001` — Open-Inspect Chrome Extension (Swarm-Ready v2)

## Summary

This v2 plan replaces the previous extension plan and is optimized for parallel AI-agent
execution.  
It fixes the earlier shortcomings by making dependencies acyclic and explicit, splitting large work
items into atomic beads, adding deterministic validation commands per bead, and specifying
security/rollout defaults so implementers do not need to make product or architecture decisions.

## How This Plan Fixes Prior Shortcomings

1. Dependency contradictions are removed with a strict DAG and batch schedule.
2. Oversized beads are split into 24 atomic beads with narrow file ownership.
3. Manual-only acceptance is replaced by deterministic commands and test gates.
4. Security and rollout details are explicit (JWT claims, TTL, issuer/audience, packaging, managed
   install path).
5. Swarm collision risk is reduced by a file-ownership matrix and conflict-domain sequencing.

## Public APIs / Interfaces / Type Changes

1. Add extension auth env contract:
   - `EXTENSION_JWT_SECRET` (required)
   - `EXTENSION_JWT_ISSUER` default `open-inspect-web`
   - `EXTENSION_JWT_AUDIENCE` default `open-inspect-extension`
2. Add web extension endpoints under `/api/extension/*`:
   - `POST /api/extension/token`
   - `GET /api/extension/me`
   - `GET /api/extension/repos`
   - `POST /api/extension/sessions`
   - `POST /api/extension/sessions/:id/prompt`
   - `POST /api/extension/sessions/:id/ws-token`
3. JWT claims contract for extension token:
   - `sub`, `login`, `name`, `email`, `iat`, `exp`, `iss`, `aud`, `jti`
   - TTL fixed to 8h for MVP.
4. WS client message contract update in control-plane/shared types:
   - `prompt` message accepts optional `source?: MessageSource`.
   - Runtime default remains `"web"` when absent.
5. Extension context attachment contract:
   - Attachment format uses existing attachment object with
     `mimeType: "application/vnd.openinspect.extension-context+json"`.
   - Payload is JSON string in `content`.
   - Serialized hard cap: `65536` bytes.
   - Deterministic truncation metadata required: `truncated`, `originalBytes`, `finalBytes`,
     `strategy`.
6. Prompt source attribution guarantees:
   - HTTP extension route always injects `source: "extension"`.
   - WS prompt can carry `source: "extension"` and persists in message rows/logs.

## Test Cases and Scenarios

1. Unit: extension JWT verify fails on missing `aud`, `iss`, `exp`, and expired `exp`.
2. Unit: deterministic truncation always produces same output and metadata for same input.
3. Unit: context attachment schema rejects invalid mimeType and malformed JSON.
4. Integration: WS `prompt` with `source: "extension"` persists source in message row.
5. Integration: WS `prompt` without source still persists `"web"`.
6. Integration: HTTP extension prompt route always forwards `source: "extension"`.
7. Integration: all `/api/extension/*` routes reject invalid bearer with 401.
8. E2E: `/extension/connect` token mint and `/api/extension/me` round trip.
9. E2E: side panel can load repos, create session, send prompt, and stream events.
10. E2E: non-React page capture returns useful DOM payload.
11. E2E: React page capture includes React metadata payload.
12. E2E: screenshot permission denied still allows prompt with non-image context.
13. Regression: existing web prompt route at `/api/sessions/[id]/prompt` still reports
    `source: "web"`.
14. Regression: Slack and GitHub prompt flows remain unchanged.

## Explicit Assumptions and Defaults

1. Scope remains extension-specific; no new Slack/web statistics feature is added in this epic.
2. Auth flow remains explicit token handoff via `/extension/connect` for MVP.
3. Token TTL is fixed at 8h; revocation list is out of scope for this epic.
4. Default rollout is managed install first; Chrome Web Store publishing is out of scope.
5. Extension host permissions are broad internal for MVP; domain allowlist tightening is post-MVP.
6. Screenshot capture is opt-in and off by default.
7. DOM/React structured context is primary path; screenshot is strictly fallback.
8. Attachment DB storage format remains JSON string; no schema migration is introduced.
9. PR-creation parity from extension sessions is unchanged and not expanded in this epic.
10. Terraform changes are limited to production environment files under
    `/Users/ben/code/poolside/agentic/background-agents/terraform/environments/production/`.
